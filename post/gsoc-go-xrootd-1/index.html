(GSoC) Pure-Go client for XRootD: Weeks 1 & 2

Jun 11, 2018
2 min read
Last updated on Jun 11, 2018

I am happy to announce that this summer I am working on the GSoC 2018 project titled “Pure-Go client for XRootD”.
This project is a part of Go-HEP project which brings a set of libraries and applications for High Energy Physicists written in the Go programming language.

XRootD is a protocol for access to the clustered data repositories and I am going to implement a client for that protocol and FUSE-based file system using that client.

So far, I have implemented such features:

  • An encoding and decoding of the request and response structures. So, it’s possible to write protocol.Marshal(streamID, requestID, request) and receive bytes which can be used to send a request. (Already merged!)
  • A multiplexer that uses channels to communicate between goroutines (Go language feature, think of them as very lightweight threads, if you don’t know exactly). There is a one goroutine that collects responses from the server and then routes them to the awaiting request calls. (It is already merged!)
  • A Client that connects to the server and exchanges a handshake. (It is already merged!)
  • A protocol request which is used to obtain server protocol version and security requirements. These requirements will determine whether request needs to be signed or not. (Review is in progress)
  • Basics of request signing. Before each request which needs to be signed a special request is sent. That special request tells the server that “the next request will have that request id and its hash will equal to that value”. (I’m still refactoring that one, so no PR yet)

To be honest, I’m a bit behind schedule because I thought that I’ll have more exams and study duties during the next 2 weeks and not these ones. But, I have finished most of my studies at university, so I expect to make more significant progress during upcoming 2 weeks than I planned before.
