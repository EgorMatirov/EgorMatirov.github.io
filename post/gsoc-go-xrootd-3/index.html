(GSoC) Pure-Go client for XRootD: Weeks 5 & 6

Jun 26, 2018
1 min read
Last updated on Jun 26, 2018

I am happy to announce that this summer I am working on the GSoC 2018 project titled “Pure-Go client for XRootD”.
This project is a part of Go-HEP project which brings a set of libraries and applications for High Energy Physicists written in the Go programming language.

XRootD is a protocol for access to the clustered data repositories and I am going to implement a client for that protocol and FUSE-based file system using that client.

During these 2 weeks I have worked on such features:

  • Support of the krb5 (Kerberos) authentication protocol. It’s much more secure than previously implemented “unix” authentication and can be used in real-life scenarios.
  • Support for the redirections. They are used when a file is not stored on the current server but it knows where to redirect client. Adding this feature makes it possible to copy files from the eospublic.cern.ch, yay!

As for now, I am working on improving redirection support and implementing support for the GSI authentication protocol. I have already managed to configure the XRootD server to use the GSI and currently working on implementing authentication process in Go.
